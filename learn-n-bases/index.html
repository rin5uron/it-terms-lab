n進数とは「n個の数字」で数を数える方法です。普段使う10進数は10個、2進数は2個の数字を使います。

n進数とは？

n進数 = 「n個の数字を使って数を表す方法」

例えば...
  • 10進数： 0〜9 の10個を使う
  • 2進数 ： 0, 1 の2個だけを使う
  • 16進数： 0〜9, A〜F の16個を使う

桁上がりのルール

使える数字を全部使い切ったら、次の桁に進む！

  • 10進数：9の次は10（10で桁上がり）
  • 2進数 ：1の次は10（2で桁上がり）
  • 16進数：Fの次は10（16で桁上がり）

「基数」は使う数字の個数、「桁の重み」は各桁の重みを表します。これらを使ってn進数を10進数に変換できます。

基数・桁の重み

基数
：n進数のn。１桁で使える数字の種類数。例：２進数なら０と１の２種類を使って表すので基数は２。

桁の重み
：各桁が表す量を決める倍率。基数に各桁の位置を表す指数をかけたもの。

例：10進数 1234 の場合
1234 = 1×10³  + 2×10²  + 3×10¹  + 4×10⁰
     = 1×1000 + 2×100  + 3×10   + 4×1
     = 1000   + 200    + 30     + 4
     = 1234（答え）

各桁の重みの概念

  • 千の位（1）： 重みは10³ = 1000
  • 百の位（2）： 重みは10² = 100
  • 十の位（3）： 重みは10¹ = 10
  • 一の位（4）： 重みは10⁰ = 1

-なぜ指数が0から始まるの？-

  • 一の位は「何倍にもしていない」= 1倍 = 基数⁰
  • 十の位は「基数を1回かけた」= 基数¹
  • 百の位は「基数を2回かけた」= 基数²
桁の重み は一の位から基数の0乗、1乗、2乗...と増えていく！

人間(10進数)、PC(2進数)、開発者(16進数)のスムーズな連携のため、基数変換は必須の知識です。

基数変換が試験に出る理由

コンピュータは日々、主に10進数と16進数を２進数に変換して計算しているから

なぜ 10・2・16進数の相互変換が必要なのか

詳しく見る

10進数

人間が普段使う表記

2進数

コンピューターが内部で使う表記（オン/オフ）

16進数

2進数（4ビット）= 16通り → 相性抜群！

16進数の1桁 = 2進数の4桁に対応する。変換が簡単かつ2進数４桁を１桁でシンプルに表せる！

例：2進数の１０１０→16進数のA
例：16進数のA3 → 2進数の1010 0011

実際の変換例

色コード #FF0000 を2進数に変換
#FF0000 = FF + 00 + 00 FF = 11111111 (2進数) 00 = 00000000 (2進数) 00 = 00000000 (2進数) 結果：11111111 00000000 00000000 （赤255、緑0、青0 = 赤色）
メモリアドレス 0x1A2B を2進数に変換
0x1A2B = 1 + A + 2 + B 1 = 0001 (2進数) A = 1010 (2進数) 2 = 0010 (2進数) B = 1011 (2進数) 結果：0001 1010 0010 1011

PC内部では2進数で処理され、開発者は16進数で確認し、私たちは10進数で結果を見ます。

実用のイメージ

コンピューターの数値処理の流れ

2進数（内部処理）
↓ 4ビット単位で変換
16進数（エンジニアが見るアドレスやデータ）
↓ 人に見せるときや計算

例えば、プログラムでメモリの場所を確認するときは16進数で表示されることが多く、それを人間が理解しやすいように10進数に変換したり、コンピューターの内部動作を理解するために2進数に変換したりする。

基数変換の方法

10進数 → n進数

（1）整数部の変換

nで割って、余りを下から順に並べる！
基数nで割ることで、桁ごとの文字列を求めている!
例：13 を 2進数に変換する場合
13 ÷ 2 = 6 ... 1
6 ÷ 2 = 3 ... 0
3 ÷ 2 = 1 ... 1
1 ÷ 2 = 0 ... 1

→ 逆順に読む → 1101（答え）

（2）小数部の変換

基数nを掛けて、出てきた整数部分を左から順に読む！
基数nで割ることで、桁ごとの文字列を求めている
例：0.625 を 2進数に変換する場合
0.625 × 2 = 1.25 → 1
0.25 × 2 = 0.50 → 0
0.50 × 2 = 1.00 → 1

→ 左から読む → 0.101（答え）

n進数 → 10進数
各桁に桁の重みをかけて、それぞれを合計する！ 桁の重みをかけると、整数部分が少数部分の一の位になる
例1：2進数 1101 を 10進数に変換
    1×2³ + 1×2² + 0×2¹ + 1×2⁰
       = 8   + 4   + 0   + 1   
       = 13（答え）
例2：16進数 1A を 10進数に変換
    1×16¹ + A×16⁰ = 1×16 + 10×1 = 26
    （※A=10、B=11、... F=15）

迷ったら10進数 → 10進数に変換してみて

例1：nで割る方法（10進数498を10進数に直す）
    498 ÷ 10 = 49 ... 8
     49 ÷ 10 =  4 ... 9
      4 ÷ 10 =  0 ... 4
    
    → 逆順に読む → 498（答え）
例2：nを掛ける方法（10進数0.953を10進数に直す）
    0.953 × 10 = 9.53 → 9
    0.53  × 10 = 5.30 → 5
    0.30  × 10 = 3.00 → 3
    
    → 左から読む → 0.953（答え）
例3：桁の重みをかける方法（498を10進数に直す）
    4×10² + 9×10¹ + 8×10⁰
        = 4×100 + 9×10 + 8×1
        = 400   + 90   + 8 
        = 498（答え）

パターンを知り、効率よく解けるようにしましょう。
