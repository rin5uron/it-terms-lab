問1：基本的なHTTP通信（令和4年サンプル問）

PCがWebサーバにHTTPで接続を開始する際のTCPパケットでは、送信元ポート番号が50001、宛先ポート番号が80であった。このWebサーバからPCへの戻りのTCPパケットでの、送信元ポート番号と宛先ポート番号の組合せはどれか。

正解と解説

送信元ポート番号：80、宛先ポート番号：50001

【解説】

戻りの通信では、送信元と宛先のポート番号が逆になります。

行き（リクエスト）
送信元: PC (50001)
宛先: Webサーバ (80)
帰り（レスポンス）
送信元: Webサーバ (80)
宛先: PC (50001)

したがって、Webサーバからの戻りのパケットは、送信元が「80」、宛先が「50001」となります。

問2：プロキシ経由の通信（令和元年秋 問34）

クライアントAが、ポート番号8080のHTTPプロキシサーバBを経由して、ポート番号80のWebサーバCにアクセスする。このとき、宛先ポート番号が8080になるTCPパケットはどれか。

ア：AからBへのHTTP要求
イ：AからBへのHTTP要求 と BからCへのHTTP要求
ウ：AからBへのHTTP要求 と CからBへのHTTP応答
エ：BからCへのHTTP要求 と CからBへのHTTP応答

正解と解説

ウ：AからBへのHTTP要求 と CからBへのHTTP応答

【解説】

宛先ポート番号が「8080」になるのは、通信の受け手が「プロキシサーバB」であるパケットです。

1. A → B へのHTTP要求
AがBにリクエストを送るため、宛先はBのポート「8080」。
2. B → C へのHTTP要求
BがCにリクエストを中継するため、宛先はCのポート「80」。
3. C → B へのHTTP応答
CがBに応答を返すため、宛先はBのポート「8080」。
4. B → A へのHTTP応答
BがAに応答を中継するため、宛先はAのポート（例:50001など）。

したがって、宛先ポートが8080になるのは「AからBへのHTTP要求」と「CからBへのHTTP応答」です。
